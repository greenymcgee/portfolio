## Description

Add the visible content layer of the edit page: the auto-focused title input, the editor area styles, and the publishedAt subtitle. No new state — these components receive state from `EditPostForm` (PR 7) as props.

**`TitleInput`:**
- Auto-focused on mount.
- Styled to be invisible: no border, no background — visually integrated with the page content.
- On change, updates `title` state in `EditPostForm`, which triggers the autosave debounce.
- On unique-constraint error from autosave, renders an inline error below the input.

**`PublishedAtSubtitle`:**
- Static `<time>` element set once on page load using the existing `Time` component.
- Draft posts: shows current date/time captured at render time.
- Published posts: shows the actual `publishedAt` value from the DB.
- Format: `"MMMM do, yyyy 'at' h:mm a"` — e.g. `May 3rd, 2026 at 10:30 AM`

**Editor area styles:**
- Style the content area to match `design-reference.png`.
- Design reference is a guide, not a pixel-perfect requirement (constraint 3).

**Inline error states (deferred from PR 7):**
- `state.threwUniqueConstraintError` renders a hard-coded error message below `TitleInput`.
- `state.dtoError?.fieldErrors?.content` renders inline errors near the editor.

## Technical Notes

- Reuse the existing `Time` component for `PublishedAtSubtitle` — do not implement date formatting from scratch.
- `PublishedAtSubtitle` is static (no state), set once at render time. It does not update after publish/unpublish — the page is navigated away from on publish; unpublish is in-place but the subtitle is considered a render-time snapshot. See `decisions.md` → D15.
- Design reference: `docs/discovery/edit-post/inputs/design-reference.png`. Design is a guide only.
