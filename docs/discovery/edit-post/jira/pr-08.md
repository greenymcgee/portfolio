## Description

Add the sticky action bar and wire up the Lexical editor. This PR also introduces the new `RichTextEditor` (which replaces the renamed `LegacyRichTextEditor` in name only — it is a different component with different responsibilities) and lifts the `LexicalComposer` to `EditPostClient`.

**`LexicalComposer` strategy (critical):**
The `ToolbarPlugin` must be a descendant of a `LexicalComposer`. Because the toolbar renders in `ActionBar` (above the editor DOM subtree), the `LexicalComposer` must be owned by their common ancestor: `EditPostClient`. `RichTextEditor` is rendered with no internal composer; it receives the composer context from the ancestor. See `decisions.md` → D5.

```
EditPostClient ('use client')
  └── LexicalComposer         ← owned here
        ├── ActionBar (sticky)
        │     ├── ToolbarPlugin   ← inside LexicalComposer ✓
        │     └── SaveStateIndicator
        └── RichTextEditor (new — no internal LexicalComposer)
```

**New `RichTextEditor`:**
- No internal `LexicalComposer` (it assumes one exists in its ancestor tree).
- No embedded toolbar (toolbar lives in `ActionBar`).
- `ToolbarPlugin` is re-exported from `globals/components/richTextEditor/index.ts` for use in `ActionBar`.

**`ActionBar`:**
- Sticky top bar.
- Contains: `ToolbarPlugin`, `SaveStateIndicator`, and placeholder slots for buttons (PRs 10–11).
- Receives `cancelPendingDebounce` and `flushPendingDebounce` from `EditPostClient` as props (passed through to buttons in later PRs).

**`SaveStateIndicator`:**

| State | Display |
|-------|---------|
| `idle` | Nothing |
| `saving` | Spinner only |
| `saved` | "Saved" — persists until next save cycle |
| `error` | Inline error text |

Autosave errors route through the indicator. Sonner is reserved for publish/unpublish/close failures.

## Technical Notes

- `LegacyRichTextEditor` (renamed in PR 5) creates its own `LexicalComposer` internally — that behavior is unchanged for its existing consumers. The new `RichTextEditor` does not.
- The `ToolbarPlugin` re-export must not break existing `LegacyRichTextEditor` consumers.
- See `decisions.md` → D5 (LexicalComposer strategy) and D14 (SaveStateIndicator / Sonner split).
