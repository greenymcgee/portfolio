## Description

Implement the core edit page structure without the UI: the route, the data-fetching RSC, the client shell, the autosave hook, and the auth guard.

**Component hierarchy:**

```
app/posts/[id]/edit/page.tsx      sync RSC — auth guard + <Suspense>
  └── EditPostContent             async RSC — calls getPost (cached); passes post to client
        └── EditPostClient        'use client' — owns title/description/content state + useAutoSave
```

**Auth guard:** Two-layer match of the existing `CreatePostForm` pattern:
1. `page.tsx` (sync RSC): calls `authenticateAPISession()`; `redirect(ROUTES.login)` if no session.
2. `EditPostClient` (client component): `useLayoutEffect` checks session on mount; redirects if missing. Belt-and-suspenders against client-side navigation without a session.

**`useAutoSave` hook:**
```ts
useAutoSave({
  fields: { title, description, content },
  delay: 1000,
  onSave: (fields) => startTransition(() => updatePost(fields)),
})
// Returns: { cancelPendingDebounce, flushPendingDebounce }
```
Uses `useRef` + `setTimeout`/`clearTimeout`. `startTransition` keeps autosave non-blocking. No new dependency. See `decisions.md` → D4.

**Autosave state machine** (tracked in `EditPostClient`, drives `SaveStateIndicator` in PR 9):

| State | Transition |
|-------|-----------|
| `idle` | Initial; no autosave has fired this session |
| `saving` | `onSave` fires |
| `saved` | `onSave` completes successfully |
| `error` | `onSave` returns error result |

`cancelPendingDebounce` and `flushPendingDebounce` are passed down to action bar buttons (PRs 10–11).

## Acceptance Criteria

- [ ] `app/posts/[id]/edit/page.tsx` exists as a sync RSC with a `<Suspense>` wrapper
- [ ] Unauthenticated requests to `/posts/[id]/edit` redirect to login (server-side)
- [ ] `EditPostClient` redirects on mount if session is missing (`useLayoutEffect`)
- [ ] `useAutoSave` debounces 1 second after the last field change
- [ ] `cancelPendingDebounce` prevents the pending save from firing
- [ ] `flushPendingDebounce` calls `onSave` immediately with current fields
- [ ] `startTransition` wraps the `updatePost` call in `onSave`
- [ ] Autosave state transitions: `idle` → `saving` → `saved` / `error`
- [ ] `pnpm validate` passes

## Technical Notes

- Auth guard pattern: match `features/posts/components/createPostForm/createPostForm.tsx`.
- `EditPostClient` is initialized from the `post` prop fetched by `EditPostContent`. These values are the initial state only — they are never written back after mount.
- All mutable form state (`title`, `description`, `content`) lives in `EditPostClient` and is passed to children as props.
- See `decisions.md` → D5 (LexicalComposer strategy — LexicalComposer is added in PR 9, not here) and D16 (auth guard).
