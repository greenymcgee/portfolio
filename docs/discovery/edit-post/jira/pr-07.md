## Description

Implement the core edit page structure without the UI: the route, the data-fetching RSC, the
client shell, the autosave wiring, and the auth guard.

**Component hierarchy:**

```
app/posts/[id]/edit/page.tsx      sync RSC — auth guard + <Suspense>
  └── EditPostContent             async RSC — calls getPost (cached); passes post to client
        └── EditPostClient        'use client' — owns title/description/content state + autosave
```

**Auth guard:** Two-layer match of the existing `CreatePostForm` pattern:
1. `page.tsx` (sync RSC): calls `authenticateAPISession()`; `redirect(ROUTES.login)` if no session.
2. `EditPostClient` (client component): `useLayoutEffect` checks session on mount; redirects if
   missing. Belt-and-suspenders against client-side navigation without a session.

**Autosave (`useActionState` + inline debounce):**

```ts
const formRef = useRef<HTMLFormElement>(null)
const timeoutRef = useRef<ReturnType<typeof setTimeout> | null>(null)

const [state, updateAction, pending] = useActionState(
  updatePost,
  { status: 'IDLE' } as UpdatePostState,
)

const cancelDebounce = () => {
  if (timeoutRef.current) clearTimeout(timeoutRef.current)
  timeoutRef.current = null
}

const flushDebounce = () => {
  cancelDebounce()
  if (formRef.current) updateAction(new FormData(formRef.current))
}
```

On each field change: `cancelDebounce()`, then schedule a 1-second timeout. When it fires,
`updateAction(new FormData(formRef.current))`. No custom hook. See `decisions.md` → D31.

**Autosave display** (all derived from `state` and `pending`, no mirrored state):

| Condition | Display |
|-----------|---------|
| `state.status === 'IDLE'` | Nothing |
| `pending` | `SaveStateIndicator` spinner |
| `!pending && state.status === 'SUCCESS'` | `SaveStateIndicator` → "Saved" |
| `!pending && state.status === 'ERROR'` | `SaveStateIndicator` error |
| `state.threwUniqueConstraintError` | `TitleInput` hard-coded error message |
| `state.dtoError?.fieldErrors?.content` | Inline errors near editor |

`cancelDebounce` and `flushDebounce` are passed as props to action bar buttons (PRs 10–11).

## Technical Notes

- Auth guard pattern: match `features/posts/components/createPostForm/createPostForm.tsx`.
- `EditPostClient` is initialized from the `post` prop fetched by `EditPostContent`. These values
  are the initial state only — they are never written back after mount.
- All mutable form state (`title`, `description`, `content`) lives in `EditPostClient` and is
  passed to children as props.
- See `decisions.md` → D5 (LexicalComposer strategy — LexicalComposer is added in PR 9, not here)
  and D16 (auth guard).
