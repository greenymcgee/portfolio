## Description

Rename the existing `RichTextEditor` component to `LegacyRichTextEditor` and update every consumer. No behavior changes.

This rename must land before any edit page work (PRs 6–8) touches the `RichTextEditor` name, because PR 8 introduces a new `RichTextEditor` that differs from the existing one (no internal `LexicalComposer`, no embedded toolbar). Without this rename, consumers of the old component would break. See `decisions.md` → D21.

**Scope:**

- Find and update all import sites across the codebase.
- No logic, styling, or API changes.
