## Description

Implement the `PublishUnpublishButton` UI component and wire it into the `ActionBar`.

- On submit: accesses `formRef.current` to read current form values; adds hidden inputs for `id`, `title`, `description`, `content`, `publishedAt`, and (when publishing) `redirectUrl`.
- **While in flight:** button shows **"Publishing..."** or **"Unpublishing..."** with a spinner.
- **On publish success:** redirect to `ROUTES.post(id)` (handled by the action). Sonner toast — "Success!" — appears on the post page after redirect (D39).
- **On unpublish success:** Sonner toast — "Success!"; label toggles in-place. No redirect (D39).
- **On publish failure:** Sonner toast — "Post could not be published" (D39).
- **On unpublish failure:** Sonner toast — "Post could not be unpublished" (D39).

See `decisions.md` → D3 (separate actions), D9 (Publish redirects; Unpublish is in-place), D40 (toggle contract), D41 (split rationale), D42 (no disabled state).

## Acceptance Criteria

- [ ] Publish: shows "Publishing..." + spinner while in flight, redirects to post detail on success
- [ ] Unpublish: shows "Unpublishing..." + spinner while in flight, Sonner toast — "Success!" + toggles label in-place on success; no redirect
- [ ] Publish failure: Sonner toast — "Post could not be published"; label does not change
- [ ] Unpublish failure: Sonner toast — "Post could not be unpublished"; label does not change
- [ ] `pnpm validate` passes

## Technical Notes

- `PublishUnpublishButton` receives `formRef` and `cancelDebounce` as props from `EditPostForm` and calls `cancelDebounce` before submitting.
- `publishedAt` and `redirectUrl` fields are added to `UpdatePostDto` in EDIT-POST-11. The button populates them as hidden inputs from `formRef.current`.
- The button wires up `togglePostPublishedStatus` via `useActionState`. Use `withCallbacks` for Sonner toasts on success/failure.
- The button is never disabled — see `decisions.md` → D42.
- See `decisions.md` → D40 for the full toggle contract.
