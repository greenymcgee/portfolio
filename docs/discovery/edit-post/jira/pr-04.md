## Description

This is the first PR with user-visible changes. Ship after PR 2 is verified in staging.

**Three changes in one PR:**

1. **`createPost` update:** Instead of redirecting to a form, `createPost` creates a minimal draft and redirects to `ROUTES.editPost(post.id)`. The draft is created with a timestamped placeholder title (e.g. `"Untitled — 2026-05-02 10:30:45"`), a valid initial Lexical state from `createHeadlessBlogEditor()`, and an empty description.

2. **Delete `/posts/new` and `CreatePostForm`:** The form page and component are removed. `ROUTES.newPost` is removed from `globals/constants/routes.ts`; `ROUTES.editPost(id)` is added.

3. **`PostPageAdminMenuContent` update:** Adds an Edit link (`<Link href={ROUTES.editPost(post.id)}>`) and converts the New Post button from `<Link href={ROUTES.newPost}>` to `<form action={createPost}>`.

**`PostPageAdminMenuContent` after this PR:**

| Action | Implementation |
|--------|---------------|
| New Post | `<form action={createPost}>` |
| Edit | `<Link href={ROUTES.editPost(post.id)}>` (new) |
| Delete | Existing `<form action={deletePost}>` (unchanged) |

## Technical Notes

- The timestamped placeholder title (e.g. `"Untitled — 2026-05-02 10:30:45"`) is subject to the partial unique index. Two drafts created within the same second would collide — acceptable at this scale. See `decisions.md` → D1 and D8.
- `updatePost` (PR 2) must already be deployed — the edit page uses it for autosave. Ship PR 4 after PR 2 is verified in staging.
- The edit page itself does not exist yet (PR 6); clicking the Edit link from PR 4 will 404 until PR 6 lands. This is acceptable — the redirect from `createPost` is the primary path.
