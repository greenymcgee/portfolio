## Description

Add `CloseButton` to the `ActionBar`.

On submit:
1. Cancel any pending autosave debounce.
2. Submit the form. While in flight, button label changes to **"Closing..."** with a spinner.
3. **On success:** server action redirects to `ROUTES.post(id)` via `redirectPath`.
4. **On failure:** a dialog appears — **"There are unsaved changes / Are you sure you want to leave?"** — with Cancel (stays on page) and Close (destructive; a `<Link href={ROUTES.post(post.id)}>` that navigates away without saving) buttons.

See `decisions.md` → D10 (Close destination), D37 (hidden-inputs form pattern), D39 (confirmed design), D44 (redirectPath).

## Acceptance Criteria

- [ ] `CloseButton` cancels pending debounce before submitting
- [ ] `CloseButton` shows "Closing..." + spinner while in flight
- [ ] On success: server action redirects to `ROUTES.post(id)` via `redirectPath`
- [ ] On failure: "There are unsaved changes / Are you sure you want to leave?" dialog appears
- [ ] Dialog Cancel: stays on edit page
- [ ] Dialog Close (destructive): `<Link href={ROUTES.post(post.id)}>` navigates away without saving
- [ ] `pnpm validate` passes

## Technical Notes

- `CloseButton` uses `useActionState(withCallbacks(updatePost, { onSuccess }), initialState)`. Its form includes hidden inputs populated from `formRef.current` (id, title, description, content) plus a `redirectPath` hidden input set to `ROUTES.post(post.id)`. `updatePost` redirects on success when `redirectPath` is present. See `decisions.md` → D37.
- `redirectPath` is added to `UpdatePostDto` and `updatePostSchema` in EDIT-POST-11. See `decisions.md` → D44.
- The "There are unsaved changes" dialog uses the Shadcn `Dialog` from PR 2. See `decisions.md` → D39 (supersedes D13).
