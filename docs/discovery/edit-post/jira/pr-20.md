## Description

Remove `SiteNavbar` and `ClientSiteNavbar` from the root `app/layout.tsx`. Each page or layout that needs the navbar renders it directly and passes `pathname` as a server-side prop. Delete `ClientSiteNavbar` entirely.

See `decisions.md` → D47.

## Acceptance Criteria

- [ ] `ClientSiteNavbar` component and its directory are deleted
- [ ] `SiteNavbar` is removed from `app/layout.tsx`
- [ ] Pages/layouts that need the navbar render `<SiteNavbar pathname={...} />` directly with a server-side pathname
- [ ] `/login`, `/posts/[id]`, and `/posts/[id]/edit` do not render the navbar
- [ ] `AdminMenuDialog` still renders on `/posts/[id]` and `/posts/[id]/edit`
- [ ] `pnpm validate` passes

## Technical Notes

- `ClientSiteNavbar` existed solely to call `usePathname()` and forward the value to `SiteNavbar`. With server-side rendering, the pathname is available without a client hook.
- `AdminMenu` (which renders `AdminMenuDialog`) must still be reachable on post pages. The edit page's `ActionBar` or a page-level layout should render it independently of `SiteNavbar`.
