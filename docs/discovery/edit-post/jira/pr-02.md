## Description

Install the Shadcn Dialog component and split it into the project's one-component-per-directory convention under `globals/components/ui/dialog/`. This is a prerequisite for the description modal (PR 10) and the no-title confirmation dialog (PR 10).

Shadcn Dialog is built on Radix UI, which is already a transitive dependency — no new runtime dependency is introduced. See `decisions.md` → constraint 2.

**Install command:**
```bash
npx shadcn add dialog
```

After installation, split the generated output into one file per sub-component following the existing pattern in `globals/components/ui/` (e.g. `button/`, `input/`).

## Acceptance Criteria

- [ ] `globals/components/ui/dialog/` directory exists with one-per-directory structure
- [ ] `Dialog`, `DialogContent`, `DialogHeader`, `DialogTitle`, `DialogDescription`, `DialogFooter` are each in their own file (or follow whichever split matches the existing Shadcn convention in this repo)
- [ ] `globals/components/ui/dialog/index.ts` exports all sub-components
- [ ] Component renders and passes accessibility checks (focus trapping, ARIA roles)
- [ ] Dialog component tests pass
- [ ] `pnpm validate` passes

## Technical Notes

- Follow the exact same one-per-directory pattern as existing Shadcn components in `globals/components/ui/` — check what's already there and match it.
- Do not test Shadcn Dialog internals (focus trapping, a11y). Radix UI's own test suite covers those.
- Radix UI is already a transitive dependency — no `package.json` additions beyond the Shadcn-generated imports.
