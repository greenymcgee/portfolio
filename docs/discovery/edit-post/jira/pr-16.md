## Description

Replace the placeholder error branch in `EditPostContent` with a proper,
intentional error UI. After PR 14 lands, this branch handles non-not-found
errors only (DTO errors, unexpected Prisma failures, etc.), so the message
should communicate a genuine unexpected failure and give the admin a clear
recovery path.

**Current state:**
```tsx
if (error) {
  return (
    <header data-testid="edit-post-content-error">
      <h1>Edit post page</h1>
      <p className="text-destructive">Something went wrong</p>
    </header>
  )
}
```

**Goal:** A styled error component consistent with the rest of the app —
meaningful heading, plain-language explanation, and a link back to `/posts`
so the admin is never stranded.
