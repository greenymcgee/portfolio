## Description

Add the Description and Close actions to the `ActionBar`.

**`DescriptionButton` + `DescriptionModal`:**
- `DescriptionButton` lives in `ActionBar` and opens the Shadcn `Dialog`.
- `DescriptionModal` wraps a `<textarea>` for the post description.
- Description state is owned by `EditPostClient` and passed down as a prop.
- Autosave fires 1 second after the modal closes (if description changed) — the standard `useAutoSave` debounce handles this because the description field change triggers the hook.

**`CloseButton`:**

On click:
1. Cancel any pending autosave debounce.
2. Call `updatePost` directly (flush) with current field state.
3. **On success:** redirect to `ROUTES.post(id)`.
4. **On failure with no title:** show delete confirmation `Dialog`. If admin confirms deletion, delete the post and redirect to `ROUTES.posts`.
5. **On failure with a title:** Sonner error toast; admin stays on the edit page.

See `decisions.md` → D10 (Close destination) and D13 (no-title confirmation flow).

## Technical Notes

- `CloseButton` uses `flushPendingDebounce` from `useAutoSave` (exposed by `EditPostClient`), not a raw `updatePost` call with a hardcoded timeout.
- The no-title confirmation dialog uses the Shadcn `Dialog` from PR 2.
- Sonner is only used for Close failures that have a title. No-title failures use the confirmation dialog instead (deleting a no-title draft is non-destructive — it was never a real post). See `decisions.md` → D13.
