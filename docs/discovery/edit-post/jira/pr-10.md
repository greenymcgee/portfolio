## Description

Add `DescriptionButton` and `DescriptionModal` to the `ActionBar`.

**`DescriptionButton` + `DescriptionModal`:**
- `DescriptionButton` lives in `ActionBar` and opens the Shadcn `Dialog`.
- `DescriptionModal` holds temporary local state (`localDescription`) initialised from the current description value when the modal opens.
- Owns its own `useActionState(withCallbacks(updatePost, { onSuccess }), ...)`. `withCallbacks` handles the `formRef` sync and auto-close only — errors are derived from `state` directly.
- `DescriptionButton` is **disabled** when `AutoSaveStatus` is in error state.
- **Save:** form has hidden inputs (id, title, content) plus the `<textarea>` for description — submits naturally → while in flight, "Save changes" button shows inline spinner → on success `withCallbacks.onSuccess` fires: updates the description hidden input in `formRef` then closes the modal.
- **Save (failure — Zod):** `state.dtoError?.fieldErrors?.description` → bullet list in red below the "Description" label.
- **Save (failure — generic):** "Something went wrong" in red below the "Description" label.
- **Save changes** is **disabled** when description is empty.
- **Cancel:** discards temp state and closes the modal without saving. Autosave does not fire.

See `decisions.md` → D27 (modal save pattern), D37, D39 (confirmed design), D45 (formRef sync).

- [ ] "Save changes" is disabled when description is empty

- [ ] Save on success: description hidden input in `formRef` is updated to the saved value; modal closes
- [ ] Save on failure (Zod): bullet list in red below the "Description" label; modal stays open
- [ ] Save on failure (generic): "Something went wrong" in red below the "Description" label; modal stays open
- [ ] Cancel discards local description state and closes the modal without saving
- [ ] `pnpm validate` passes

## Technical Notes

- `DescriptionModal` owns its own `useActionState(withCallbacks(updatePost, { onSuccess }), ...)` — independent from the autosave action state. See `state-management.md`.
- On success, `onSuccess` must update the description hidden input in the main form before closing the modal, to prevent the next autosave from overwriting the saved description with a stale value:
  ```ts
  onSuccess: () => {
    const input = formRef.current?.elements.namedItem('description') as HTMLInputElement | null
    if (input) input.value = localDescription
    closeModal()
  }
  ```
  See `decisions.md` → D45.
