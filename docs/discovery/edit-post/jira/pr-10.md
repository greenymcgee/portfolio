## Description

Add the Description and Close actions to the `ActionBar`.

**`DescriptionButton` + `DescriptionModal`:**
- `DescriptionButton` lives in `ActionBar` and opens the Shadcn `Dialog`.
- `DescriptionModal` holds temporary local state (`localDescription`) initialised from the current description value when the modal opens.
- Owns its own `useActionState(withCallbacks(updatePost, { onSuccess }), ...)`. `withCallbacks` handles auto-close only — errors are derived from `state` directly.
- `DescriptionButton` is **disabled** when `AutoSaveStatus` is in error state.
- **Save:** form has hidden inputs (id, title, content) plus the `<textarea>` for description — submits naturally → while in flight, "Save changes" button shows inline spinner → on success `withCallbacks.onSuccess` fires and closes the modal.
- **Save (failure — Zod):** `state.dtoError?.fieldErrors?.description` → bullet list in red below the "Description" label.
- **Save (failure — generic):** "Something went wrong" in red below the "Description" label.
- **Save changes** is **disabled** when description is empty.
- **Cancel:** discards temp state and closes the modal without saving. Autosave does not fire.

**`CloseButton`:**

On submit:
1. Cancel any pending autosave debounce.
2. Submit the form. While in flight, button label changes to **"Closing..."** with a spinner.
3. **On success:** server action redirects to `ROUTES.post(id)` via `redirectPath`.
4. **On failure:** a dialog appears — **"There are unsaved changes / Are you sure you want to leave?"** — with Cancel (stays on page) and Close (destructive; a `<Link href={ROUTES.post(post.id)}>` that navigates away without saving) buttons.

See `decisions.md` → D10 (Close destination), D37 (hidden-inputs form pattern), D39 (confirmed design).

## Technical Notes

- `updatePost` gains a `redirectPath` optional DTO field (D37). `CloseButton` populates it; the server action redirects on success.
- `DescriptionModal` owns its own `useActionState(withCallbacks(updatePost, ...), ...)` — independent from the autosave action state. See `state-management.md`.
- The "There are unsaved changes" dialog uses the Shadcn `Dialog` from PR 2. See `decisions.md` → D39 (supersedes D13).
