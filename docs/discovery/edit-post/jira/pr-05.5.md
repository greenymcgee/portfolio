## Description

Add a `variant` prop to `Switch` (`globals/components/ui/switch/switch.tsx`) to
support three distinct color schemes. The `inverted` variant is required by
`UnpublishedPostsToggle` (PR 6), which renders inside `AdminMenuDialog`
(`bg-foreground` = `#d9d9d9`) — the inverse of the site's dark background.

The Shadcn `Switch` is already installed (PR #164). This PR only modifies its
color logic and introduces a `constants.ts` file (matching the `Button` pattern)
to hold the `cva` variant definitions.

| Variant | Track (off) | Track (on) | Thumb (off) | Thumb (on) |
|---------|-------------|------------|-------------|------------|
| `default` | `bg-subtle` | `bg-background` | `bg-background/70` | `bg-input` |
| `inverted` | `bg-subtle` | `bg-input` | `bg-input/80` | `bg-background` |
| `primary` | `bg-input` | `bg-primary` | existing behavior | ← same |

Both `default` and `inverted` share `bg-subtle` for the unchecked track; their
on-state track colors distinguish them. Opacity modifiers (`/70`, `/80`) on the
thumb off-state reduce visual weight so the unchecked state reads as inactive.
`primary` preserves the original Shadcn behavior in full (including dark-mode
overrides) for contexts that require a branded checked state. See
`decisions.md` → D33 and D34.

## Acceptance Criteria

- [x] `Switch` accepts `variant?: 'default' | 'inverted' | 'primary'`
- [x] `variant` defaults to `'default'` when omitted
- [x] Root element carries `data-variant` reflecting the active variant
- [x] `variant="primary"` preserves existing behavior exactly (teal track when checked, all dark-mode classes intact)
- [x] Implementation follows the `Button` pattern: `constants.ts` exports `SWITCH_TRACK_VARIANTS` and `SWITCH_THUMB_VARIANTS` via `cva`; `switch.tsx` uses `VariantProps`
- [x] `pnpm validate` passes

## Technical Notes

- The existing `size` prop is unchanged.
- Dark-mode classes on the `primary` variant must be preserved as-is — dark mode
  is a future concern but the styles should not be stripped.
- `UnpublishedPostsToggle` does not need updating in this PR; it will pass
  `variant="inverted"` in PR 6.
