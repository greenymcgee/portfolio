## Description

Add a `variant` prop to `Switch` (`globals/components/ui/switch/switch.tsx`) to
support three distinct color schemes. The `inverted` variant is required by
`UnpublishedPostsToggle` (PR 6), which renders inside `AdminMenuDialog`
(`bg-foreground` = `#d9d9d9`) — the inverse of the site's dark background.

The Shadcn `Switch` is already installed (PR #164). This PR only modifies its
color logic.

| Variant | Track | Thumb (off) | Thumb (on) |
|---------|-------|-------------|------------|
| `default` | `bg-subtle` (always) | `bg-background` | `bg-input` |
| `inverted` | `bg-input` (always) | `bg-subtle` | `bg-background` |
| `primary` | `bg-input` (off) / `bg-primary` (on) | existing behavior | ← same |

`default` and `inverted` are true inverses — track and thumb tokens swap
symmetrically. `primary` preserves the original Shadcn behavior in full
(including dark-mode overrides) for contexts that require a branded checked
state. See `decisions.md` → D33.

## Acceptance Criteria

- [ ] `Switch` accepts `variant?: 'default' | 'inverted' | 'primary'`
- [ ] `variant="default"` renders `bg-subtle` track, `bg-background` thumb off, `bg-input` thumb on
- [ ] `variant="inverted"` renders `bg-input` track, `bg-subtle` thumb off, `bg-background` thumb on
- [ ] `variant="primary"` preserves existing behavior exactly (teal track when checked, all dark-mode classes intact)
- [ ] Omitting `variant` defaults to `'default'`
- [ ] `pnpm validate` passes

## Technical Notes

- The existing `size` prop is unchanged.
- Dark-mode classes on the `primary` variant must be preserved as-is — dark mode
  is a future concern but the styles should not be stripped.
- `UnpublishedPostsToggle` does not need updating in this PR; it will pass
  `variant="inverted"` in PR 6.
