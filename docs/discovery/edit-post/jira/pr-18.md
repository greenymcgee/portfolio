## Description

Add `AutoSaveStatus` to the `ActionBar`. The component derives its display entirely from `state` and `pending` — no local state.

| State | Display |
|-------|---------|
| `idle` | "Edited [date-fns phrase]" — `formatDistanceToNow(post.updatedAt, { addSuffix: true })` |
| `saving` | Spinner + "Saving..." |
| `saved` | "Saved" — persists until next save cycle |
| `error` | "Updates not saved" — destructive red |

Autosave errors also fire a Sonner toast — "Post could not be saved".

## Acceptance Criteria

- [ ] `AutoSaveStatus` renders all four states correctly: `idle` ("Edited [date-fns phrase]"), `saving` (spinner + "Saving..."), `saved` ("Saved"), `error` ("Updates not saved" in destructive red)
- [ ] `AutoSaveStatus` receives autosave state from `EditPostForm`
- [ ] Autosave error fires a Sonner toast — "Post could not be saved"
- [ ] `pnpm validate` passes

## Technical Notes

- Use `formatDistanceToNow` from `date-fns` (already installed) for the idle phrase.
- See `decisions.md` → D14 (AutoSaveStatus / Sonner split) and D39 (confirmed design).
