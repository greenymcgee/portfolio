## Description

Add proper 404 handling to `EditPostContent` and wire up a `not-found.tsx` at
the edit route segment.

**`EditPostContent` change:**

PR #180 already updated `EditPostContent` to destructure `errorType` from
`getPost` and renders the generic error UI for any truthy `errorType`. The
remaining work in this PR is to split that check so the `'not-found'` case
calls `notFound()` from `next/navigation` (delegating to the nearest
`not-found.tsx`), while all other truthy `errorType` values continue to render
the `data-testid="edit-post-content-error"` generic error UI.

`getPost` returns `{ errorType: ActionError | 'unhandled' | null, post, status }`.
Check `errorType === 'not-found'` first, then fall through to the generic check.

**`not-found.tsx`:**
- Lives at `app/posts/[id]/edit/not-found.tsx`.
- Renders a user-facing 404 message appropriate to the edit context.
- Consistent with the visual style of the rest of the app.
