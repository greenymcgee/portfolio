## Description

Add proper 404 handling to `EditPostContent` and wire up a `not-found.tsx` at
the edit route segment.

**Prerequisite — issue #157:** `getPost` currently cannot cleanly distinguish
"post not found" from other errors. Issue #157 resolves this. PR 14 depends
on that fix before the not-found case can be handled correctly here.

**`EditPostContent` change:**
- When `getPost` signals that the post does not exist, call `notFound()` from
  `next/navigation`. Next.js delegates to the nearest `not-found.tsx`.
- All other errors continue to render the existing
  `data-testid="edit-post-content-error"` generic error UI.

**`not-found.tsx`:**
- Lives at `app/posts/[id]/edit/not-found.tsx`.
- Renders a user-facing 404 message appropriate to the edit context.
- Consistent with the visual style of the rest of the app.
