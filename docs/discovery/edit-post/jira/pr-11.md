## Description

Implement the full publish/unpublish stack: server action, DTO, service, repository method, and the `PublishUnpublishButton` UI component.

**Backend:**

- `PublishPostDto` validates `{ id, publishing: boolean, title, description, content }`.
- `PostService.publish` calls `authenticateAPISession()` then `authorizeUser(token, 'posts', 'publish')`.
  - When `publishing: true`: validates title, description, and content are non-empty before proceeding; returns `ValidationError` if any are empty.
  - When `publishing: false`: skips content validation; sets `publishedAt` to `null`.
- `PostRepository.publish` writes `{ title, description, content, publishedAt }` atomically in one DB write. `publishedAt` is either `new Date()` (publish) or `null` (unpublish).
- `publishPost` action revalidates `CACHE_TAGS.post` and `CACHE_TAGS.posts` on success.

**Why atomic:** No pre-flush of `updatePost` is required. Publishing carries all content fields in a single write, eliminating the sequential flush-then-publish round-trip and the race conditions it would introduce. See `decisions.md` → D20.

**`PublishUnpublishButton` UI:**

- **Disabled** when any of title, description, or content is empty.
- **On Publish:** cancel debounce → call `publishPost({ id, publishing: true, ...currentFormState })` → on success, redirect to `ROUTES.post(id)`.
- **On Unpublish:** call `publishPost({ id, publishing: false, ...currentFormState })` → on success, toggle label in-place. No redirect.
- **On failure (either direction):** Sonner error toast; button label does not toggle.

See `decisions.md` → D3 (separate actions), D9 (Publish redirects; Unpublish is in-place), D19 (disabled state).

## Technical Notes

- `publishPost` carries all content fields — it is atomic. Do not pre-flush `updatePost` before calling `publishPost`. See `decisions.md` → D20.
- `PublishUnpublishButton` uses `cancelPendingDebounce` from `useAutoSave` (exposed by `EditPostClient`) before calling `publishPost`.
- `posts.publish` is a separate permission from `posts.update`. See `decisions.md` → D3.
